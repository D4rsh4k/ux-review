Three verification modes

Code alone isn't enough

Code analysis misses visual issues. Visual inspection misses implementation flaws. Neither catches usability problems. All three modes are required.

i

Code Analysis

Reads source files to check structure, design system compliance, patterns, error handling, and state coverage. Every finding includes a file:line reference.

ii

Visual Inspection

Captures screenshots at 1440px, 1024px, and 768px. Verifies rendered output, visual hierarchy, spacing, and consistency against reference features.

iii

Interactive Walkthrough

Executes every workflow end-to-end in the browser. Counts clicks, triggers errors, tests boundary cases, and follows cross-feature transitions.

Workflow

Six phases, zero guesswork

Each review follows a structured process from discovery to PR comment. Scope is confirmed before any review time is invested.

0

Discovery

Maps the feature tree from the branch diff. Identifies components, user flows, step sequences, and the closest existing feature for comparison.

1

Clarification

Confirms scope with you. Asks about focus areas, constraints, and data availability before investing review time.

2

Analysis

Reads the codebase, sets up test data, then executes every workflow live. Tests happy paths, error paths, and boundary cases.

3

Review

Evaluates across 10 dimensions. Every finding is classified by verification type and severity with file:line or screenshot evidence.

4

Report

Verdict and score first, then action items, score card, walkthrough summary, state coverage matrix, and detailed findings.

5

Post to PR

Automatically posts the review as a GitHub PR comment with the verdict visible up front and the full report in a collapsible section.

Review criteria

Ten dimensions, every angle

Each dimension has specific verification methods, a maximum severity cap, and strict rules for classification. No guessing.

1

UI Consistency Critical

Design system compliance. Catches native HTML replacing components, hardcoded colors, duplicated custom components.

2

Design Smells Critical

Anti-pattern scan: multiple primary buttons, blank panels, silent errors, missing loading states, spacing drift.

3

Error Handling Critical

Silent failures, unguarded destructive actions, missing validation. Tested both in code and live in the browser.

4

Journey Completeness Critical

State coverage: loading, empty, error, success, partial data, boundaries. Produces a state coverage matrix.

5

Heuristic Evaluation Major*

Nielsen's 10 heuristics: visibility, match, control, consistency, error prevention, recognition, flexibility, aesthetics.

6

Visual Hierarchy Critical

Gestalt principles: proximity, similarity, reading order, figure-ground. Verified from screenshots, not guessed from code.

7

Cross-Feature Consistency Major

Compares against a reference feature for patterns, state handling, component choices, and navigation.

8

Responsive Major

Layout at 1440, 1024, 768px. Overflow, interactive elements at tablet, scroll strategies. Verified visually.

9

Content & Microcopy Major

Button labels, error messages, placeholders, terminology consistency, confirmation dialog clarity.

10

Efficiency Major

Click counts, redundant round-trips, cognitive load, smart defaults. Measured live during walkthrough.

Get started

Install in seconds

Add the plugin from GitHub, then run it on any feature branch with UI changes.

1

Add the plugin source

Register the GitHub repository as a plugin marketplace in Claude Code:

> /plugin marketplace add D4rsh4k/ux-review Copied!
2

Install the plugin

Install it into your Claude Code environment:

> /plugin install ux-review Copied!
3

Run a review

On any feature branch with UI changes, invoke the skill:

> /ux-review Copied!

Alternative: local install

Clone the repo and point Claude Code to it directly:

> git clone https://github.com/D4rsh4k/ux-review.git
> claude --plugin-dir ./ux-review Copied!
