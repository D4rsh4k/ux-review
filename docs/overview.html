/ux-review

Automated UX reviews that read your code, capture screenshots, and execute workflows end-to-end — catching issues before they reach users.

Overview

What It Does

A senior UX engineer in your terminal. It reviews the actual implemented UI — not design files — by combining code analysis with live browser testing via Claude in Chrome.

Key principle: Code analysis alone can't catch visual issues. Visual inspection alone can't catch implementation issues. And neither can catch usability problems. All three modes are required.

🔍

Code Analysis

Reads source files to check structure, patterns, design system usage, and implementation correctness.

👁

Visual Inspection

Captures screenshots at multiple viewports to verify rendered output, layout, and visual hierarchy.

🖱

Interactive Walkthrough

Executes every workflow end-to-end — clicking buttons, filling forms, triggering errors, counting steps.

Review Criteria

10 Review Dimensions

Each dimension has specific verification methods and a maximum severity cap.

1

UI Consistency max: Critical

Design system compliance. Catches native HTML replacing storybook components, hardcoded colors instead of tokens, duplicated custom components.

2

Design Smells max: Critical

Anti-pattern scan: multiple primary buttons, blank panels, silent errors, missing loading states, spacing drift, typography soup.

3

Error Handling max: Critical

Silent failures, unguarded destructive actions, missing validation, unhandled rejections. Tested both in code and live.

4

Journey Completeness max: Critical

State coverage: loading, empty, error, success, partial data, boundaries. Produces a state coverage matrix.

5

Heuristic Evaluation max: Major*

Nielsen's 10 heuristics: visibility, match, control, consistency, error prevention, recognition, flexibility, aesthetics, recovery, help.

6

Gestalt & Visual Hierarchy max: Critical

Proximity, similarity, reading order, figure-ground. Visually verified from screenshots, not guessed from code.

7

Cross-Feature Consistency max: Major

Compares against a reference feature for patterns, state handling, component choices, visual density, and navigation.

8

Responsive max: Major

Layout at 1440, 1024, 768px. Overflow, interactive elements at tablet, scroll strategies. Verified visually and interactively.

9

Content & Microcopy max: Major

Button labels, error messages, placeholders, terminology consistency, confirmation clarity.

10

Efficiency max: Major

Click counts, redundant round-trips, cognitive load, smart defaults. Measured live during walkthrough.

Severity

Two-Tier Severity Rubric

Strict, objective rules. No guessing severity.

Critical

Structural DS violation, data loss, blocked workflow

Auto-fail

Major

Functional gap, missing state, UX heuristic failure

-5 points

Minor

Cosmetic, subjective, low-impact

-2 points

Info

Observation or suggestion

No deduction

Design System is the Contract

Structural storybook deviations (wrong component, duplicated component) are always Critical. Token violations (hardcoded colors) are always Major. Regardless of which dimension discovers them.

Error Prevention is Always Critical

Missing validation on user input or missing confirmation on destructive actions is Critical, regardless of dimension. Overrides per-dimension severity caps.

Scoring

How Scoring Works

Start at 100 points. Any Critical finding = auto-fail. Otherwise, deduct Major (-5) and Minor (-2) findings.

90-100 Approve
80-89 Fix & Merge
< 80 Rework

Subjective or judgment-only findings with no code evidence are flagged for designer review and don't affect the score.

Usage

How to Use It

Prerequisites

  • ✓ Claude Code (CLI, desktop app, or IDE extension)
  • ✓ Claude in Chrome extension installed and connected
  • ✓ Dev server running on localhost
  • ✓ Feature branch with UI changes

Run It

In your terminal or IDE, type: /ux-review followed by the feature name.

The skill will map your feature, ask for confirmation, then run the full review — reading code, launching the browser, executing workflows, and posting findings to your PR.

What It Checks Against

  • ✓ Your storybook component library (structural compliance)
  • ✓ CSS design tokens (color, spacing, typography)
  • ✓ The closest existing feature (cross-feature consistency)
  • ✓ Nielsen's 10 usability heuristics
  • ✓ Gestalt principles (visual hierarchy, proximity, similarity)
  • ✓ Responsive behavior at 3 viewport widths
